# 真實安全驗證與安心茶餐廳介面設計規格

## 1. 目標

在「智學手機」現有 React、Vite、TypeScript 與 `LessonEngine` 架構內，提升第二關及第三關的真實操作感：

- 第二關教授三種常見驗證方法：短訊六位數驗證碼、電郵驗證連結、完整 CAPTCHA。
- 第三關模擬真實手機餐廳點餐網站，以「安心茶餐廳」及六類十二款餐品完成分類、選餐、數量、購物車和確認流程。
- 所有資料均為本機固定模擬資料，不傳送短訊或電郵、不連接驗證服務、不收集身份資料，也不付款。

## 2. 共通學習模式

兩關均沿用：示範一次 → 跟着做 → 自己完成 → 溫和修正 → 成功回顧。

- `LessonEngine` 繼續管理 `demo`、`guided`、`independent`、`review`、錯誤次數、求助次數及完成時間。
- 只有使用者按下檢查、下一步或確認而未通過時，才增加錯誤次數。
- 「給我提示」、「再示範一次」、更換 CAPTCHA 及使用替代模式增加求助次數。
- 完成時間只計獨立練習階段。
- 所有主要觸控區最少 56px，焦點狀態清晰，狀態同時用顏色、圖示及文字表達。

## 3. 第二關：安全驗證

### 3.1 示範與跟做

示範頁先說明三種方法及辨認原則，跟做階段依序完成：

1. 短訊驗證碼；
2. 電郵驗證；
3. CAPTCHA 文字題；
4. CAPTCHA 圖片題。

### 3.2 短訊驗證碼

- 顯示遮蔽電話 `9••• ••67`，不提供電話輸入欄。
- 模擬六位數驗證碼，採六格輸入外觀，同時保留一個可貼上整組數字的可存取輸入欄。
- 支援完整輸入、錯誤、未填完整、重新傳送及逾時狀態。
- 重新傳送只重設本機固定驗證碼及倒數，不會發出網絡要求。

### 3.3 電郵驗證

- 顯示遮蔽電郵 `l••••@example.com`。
- 提供模擬收件箱，包含一封正確的「智學手機：確認練習帳號」及干擾郵件。
- 使用者需要先選出正確郵件，再按「確認練習帳號」。
- 選錯郵件時提示核對寄件者及標題；重新整理收件箱不作網絡要求。

### 3.4 完整 CAPTCHA

- 文字題顯示五位英數碼，避免 `0/O`、`1/I` 等容易混淆的組合，只採輕微字距和旋轉。
- 支援「換一組」及「使用文字替代題」；這些操作記作求助，不記作錯誤。
- 圖片題採本機自製 SVG 3×3 題庫，目標為巴士或交通燈。
- 選取狀態同時顯示外框、剔號、已選數量及可讀的 `aria-pressed`。
- 漏選提示「再找找看」；多選提示「其中一張不是指定物件」；不扣分、不立即換題。
- 提供非圖像文字替代模式，讓屏幕閱讀器使用者完成等值分類題。

### 3.5 獨立練習與回顧

獨立階段依序完成一組短訊、電郵及完整 CAPTCHA，以確保三種方法都真正練習。回顧頁列出：驗證碼要核對來源、電郵要核對寄件者、看不清 CAPTCHA 可更換或使用替代模式。

## 4. 第三關：安心茶餐廳

### 4.1 視覺方向

採用使用者提供的參考圖之資訊結構，但不複製品牌、標誌、名稱或餐品相片：

- 餐廳主視覺及「練習模式｜不會付款」標籤；
- 2×3 大圖片分類格；
- 橫向分類頁籤；
- 餐品名稱、簡介、價格及右側圖片；
- 底部固定購物車摘要；
- 購物車可修改數量、移除及核對總額。

### 4.2 餐廳及餐牌

餐廳名稱為「安心茶餐廳」。六個分類、十二款餐品為：

- 招牌套餐：沙嗲牛肉麵套餐 $46、早餐常餐 $42；
- 飯類：叉燒滑蛋飯 $48、咖喱雞飯 $52；
- 粉麵：魚蛋河 $38、雪菜肉絲米粉 $40；
- 三文治／多士：火腿蛋三文治 $28、奶油多士 $20；
- 小食：菠蘿油 $16、炸雞翼 $32；
- 冷熱飲品：港式奶茶 $14、檸檬茶 $14。

餐品資料放在獨立領域資料結構，包含 `id`、`categoryId`、`name`、`description`、`price` 及可存取圖片描述。圖片使用本機自製 SVG 餐品插圖。

### 4.3 學習流程

1. 選擇分類；
2. 選擇餐品；
3. 選擇數量並加入購物車；
4. 查看、修改或移除；
5. 核對總額並確認練習訂單。

跟做任務固定為選購一份叉燒滑蛋飯。獨立任務容許自由選餐，但必須經過分類、餐品詳情、購物車及訂單覆核頁才可完成。

### 4.4 錯誤處理

- 未選分類、餐品或必要選項時，提示下一步所需操作。
- 空購物車不可確認。
- 數量限制為 1 至 9；減至 0 時移除餐品。
- 返回餐牌保留購物車；離開關卡仍使用既有未完成確認對話框。
- 最後確認頁再次列出餐品、數量、小計及總額，按鈕文字必須包含「不會付款」。

## 5. 元件與資料界線

- `CaptchaLesson` 組合短訊、電郵、文字 CAPTCHA 及圖片 CAPTCHA 子元件。
- `OrderingLesson` 組合分類首頁、餐牌、餐品詳情、購物車及確認畫面。
- `src/domain/verification.ts` 保存固定挑戰與純驗證函式。
- `src/domain/ordering.ts` 保存分類、十二款餐品、購物車上限及總額函式。
- 練習輸入只存在 React 記憶體；`localStorage` 不新增電話、電郵、驗證碼或訂單欄位。

## 6. 測試及完成標準

- 單元測試驗證短訊格式、電郵選擇、文字 CAPTCHA、圖片漏選／多選，以及購物車數量與總額。
- 互動測試完整走過三種驗證及點餐的跟做、獨立與回顧流程。
- 驗證重新整理後敏感輸入不會保存。
- 在 390×844 手機、桌面手機外框及 200% 縮放下檢查沒有關鍵內容被裁切。
- 鍵盤、屏幕閱讀器標籤、焦點、語音提示及最少 56px 觸控區均可用。
- `npm run test` 和 `npm run build` 必須通過，瀏覽器控制台不可出現錯誤。
